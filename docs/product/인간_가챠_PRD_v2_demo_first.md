# PRD v2.0 — 인간 가챠

## 0. 문서 목적

이 문서는 기존 `오늘의 인간` PRD를 바탕으로, 서비스명을 **인간 가챠**로 변경하고 **시연 임팩트 우선** 관점에서 MVP 범위와 데모 흐름을 재정의한다.

이번 버전의 핵심 판단은 다음과 같다.

- 첫 화면부터 "오늘 나는 어떤 인간으로 뽑힐까?"가 보여야 한다.
- 로그인, 복잡한 친구 초대, 실제 통계보다 **뽑기 연출 + AI 카드 + 공유 피드**가 우선이다.
- 해커톤/발표 시연에서는 사용자가 이해하기 전에 설명을 듣게 만들면 약하다. 화면만 봐도 컨셉이 전달되어야 한다.
- 데모 실패를 막기 위해 **데모 모드, 고정 시드, fallback 카드**를 제품 요구사항에 포함한다.

---

## 1. 제품 개요

### 1.1 제품명

**인간 가챠**

### 1.2 한 줄 소개

사용자의 현재 상태를 몇 번의 선택으로 입력하면, AI가 인간 등급과 오늘의 미션을 뽑아주는 **B급 자기관리 가챠 서비스**.

### 1.3 핵심 콘셉트

기존 자기관리 앱은 사용자가 직접 목표를 세우고 기록해야 한다. 인간 가챠는 목표를 쓰게 하지 않고, 매일 한 번 **뽑기**로 인간 등급과 오늘의 미션을 부여한다.

사용자는 뽑기 결과로 `N ~ LEGEND` 등급의 인간 후보가 되고, 미션을 완료하면 해당 인간 뱃지를 확정 획득한다. 실패해도 벌점이 아니라 실패 칭호와 자아 파편을 얻는다.

### 1.4 제품 메시지

> 갓생을 강요하지 않는다. 오늘 하루 덜 망한 인간을 뽑는다.

> 목표를 세우는 앱이 아니라, 오늘의 목표를 뽑는 앱.

## 2. v1 진단과 v2 개선 방향

### 2.1 v1의 강점

- `자기관리 + 가챠 + B급 감성` 조합이 직관적이다.
- 후보 등급과 미션 성공 뱃지를 분리한 구조가 행동 동기를 만든다.
- 실패도 콘텐츠화한다는 차별점이 있다.
- AI 개인화가 서비스 이유와 잘 연결된다.

### 2.2 v1의 약점

| 문제 | 영향 | v2 개선 |
|---|---|---|
| Google 로그인, 친구방, 전체방, 자아 파편, 통계가 모두 P0처럼 보임 | 구현이 분산되고 핵심 재미가 늦게 나옴 | 데모용 P0와 제품용 P1/P2를 분리 |
| 데모 흐름이 로그인부터 시작함 | 발표 초반 임팩트 약함 | 첫 10초 안에 캡슐/픽업/뽑기 버튼 노출 |
| 소셜 기능이 실제 친구 초대처럼 커짐 | MVP 구현 리스크 증가 | 더미 친구방/전체방을 명시적으로 데모 피드로 처리 |
| AI 생성 실패 시 데모가 무너질 수 있음 | 발표 안정성 낮음 | 고정 데모 카드 + fallback JSON + seed draw 필수화 |
| 제품명과 핵심 은유가 분리됨 | 기억에 남는 이름이 약해짐 | 서비스명과 UX를 모두 `인간 가챠`로 통일 |

### 2.3 v2 우선순위 원칙

1. **시연에서 안 보이는 기능은 P0가 아니다.**
2. **뽑기 결과 화면이 서비스의 메인 상품이다.**
3. **AI는 기술 설명이 아니라 웃긴 개인화 결과로 증명한다.**
4. **실제 운영 정확성보다 데모 안정성이 먼저다.**
5. **친구방은 실제 소셜 네트워크가 아니라 바이럴 가능성을 보여주는 무대다.**

## 3. 문제 정의

### 3.1 사용자 문제

자기관리를 하고 싶어도 많은 사용자는 앱을 오래 쓰지 못한다.

주요 이유:

- 목표를 직접 적는 순간부터 부담스럽다.
- 기록형 자기관리 앱은 진지하고 죄책감을 만든다.
- 실패하면 남는 것이 없고 앱을 다시 켜기 싫어진다.
- 매일 앱을 열 기대감이 부족하다.
- 친구에게 공유할 만큼 웃기거나 가벼운 결과물이 없다.

### 3.2 해결 방향

인간 가챠는 자기관리를 `계획 작성`이 아니라 `하루 한 번 뽑기 이벤트`로 바꾼다.

| 기존 자기관리 앱 | 인간 가챠 |
|---|---|
| 목표를 직접 작성 | 목표를 뽑기로 부여 |
| 실패하면 죄책감 | 실패도 칭호와 드립으로 콘텐츠화 |
| 기록 중심 | 결과 카드와 뱃지 중심 |
| 진지한 성취 관리 | B급 인간 등급 수집 |
| 혼자 쓰는 루틴 | 친구방에서 구경하고 놀리는 피드 |
| 지속 동기 약함 | 매일 한 번 등급 뽑기 기대감 |

## 4. 목표와 성공 기준

### 4.1 제품 목표

- 사용자가 텍스트를 쓰지 않고 오늘의 미션을 받을 수 있게 한다.
- AI가 사용자 상태를 반영한 웃긴 인간 카드와 미션을 생성한다.
- 뽑기, 미션, 뱃지, 실패 칭호를 하나의 짧은 루프로 연결한다.
- 발표 시연에서 90초 안에 컨셉, 재미, AI 개인화, 확장 가능성이 모두 전달되게 한다.

### 4.2 데모 성공 기준

시연자는 90초 안에 아래 반응을 만들어야 한다.

| 기준 | 목표 |
|---|---|
| 이해 | 관객이 "자기관리 목표를 가챠로 뽑는 앱"이라고 바로 이해 |
| 재미 | 뽑기 결과 문구에서 한 번 이상 웃음 또는 반응 발생 |
| 기술성 | AI가 사용자 상태를 반영해 카드가 달라진다는 점 전달 |
| 완성도 | 뽑기 → 미션 → 뱃지 → 친구방 공유까지 끊기지 않음 |
| 기억성 | 발표 후 서비스명을 `인간 가챠`로 기억 가능 |

### 4.3 MVP 성공 기준

- 앱 실행 후 10초 안에 인간 가챠의 핵심 화면이 보인다.
- 온보딩은 4개 질문 이하, 30초 이내 완료 가능하다.
- 오늘 상태 선택 후 1번의 버튼으로 뽑기가 진행된다.
- 뽑기 연출은 최소 3단계로 결과 기대감을 만든다.
- AI 또는 fallback으로 카드가 5초 이내 생성된다.
- 미션 완료 시 뱃지 획득 화면이 나온다.
- 친구방/전체방에는 발표용 피드가 자연스럽게 표시된다.
- 데모 모드에서 원하는 등급과 결과를 안정적으로 재현할 수 있다.

## 5. 타깃 사용자

### 5.1 Primary Target

**20대 대학생, 취준생, 주니어 개발자**

특징:

- 자기관리를 해야 한다는 압박은 있지만 오래 지속하지 못한다.
- 목표 작성과 긴 기록을 귀찮아한다.
- 등급표, 뽑기, 밈, B급 문구에 반응한다.
- 친구와 결과를 공유하고 가볍게 놀리는 경험에 익숙하다.
- AI 서비스에 호기심이 있고, 개인화 결과를 재미로 소비한다.

### 5.2 Secondary Target

- 시험 준비생
- 직장인
- 루틴을 만들고 싶은 일반 사용자
- 친구들과 가벼운 챌린지를 하고 싶은 사용자

## 6. 핵심 가치 제안

### 6.1 사용자 가치

- 목표를 직접 쓰지 않아도 된다.
- 오늘의 상태만 고르면 미션이 자동으로 나온다.
- 미션이 작고 웃겨서 부담이 낮다.
- 높은 등급이 나오면 하루가 이벤트처럼 느껴진다.
- 실패해도 웃긴 칭호와 자아 파편이 남는다.
- 친구방에서 내 결과가 공유되어 참여 동기가 생긴다.

### 6.2 차별점

- 목표 작성형 자기관리 앱이 아니라 **목표 뽑기형 자기관리 앱**이다.
- AI가 매일 다른 인간 카드, 미션, 성공 뱃지, 실패 칭호를 생성한다.
- 뽑기 결과와 미션 완료 보상을 분리해 기대감과 행동 유도를 동시에 만든다.
- 실패를 감점이 아니라 콘텐츠로 만든다.
- 친구방/전체방을 통해 "혼자 하는 자기관리"를 "구경 가능한 자기관리"로 바꾼다.

## 7. Demo-First 제품 원칙

### 7.1 첫 30초 원칙

사용자는 앱을 실행하고 30초 안에 아래를 모두 봐야 한다.

1. 오늘의 픽업 인간
2. 확률 버프
3. 오늘 상태 선택
4. 인간 가챠 뽑기 버튼
5. 뽑기 연출 시작

### 7.2 Screenshotable Moment 원칙

핵심 화면은 캡처했을 때 바로 공유 가능한 카드처럼 보여야 한다.

필수 screenshotable moment:

- 홈: 오늘의 픽업 인간
- 뽑기: 인간성 스캔 중
- 결과: `SR 계획적인 취준 인간 후보`
- 성공: `도망은 안 간 사람` 뱃지
- 친구방: 친구들의 리액션 피드

### 7.3 데모 안정성 원칙

해커톤/발표 환경에서는 네트워크, AI 응답, 인증 실패가 발생할 수 있다. 따라서 아래를 필수로 둔다.

- Demo Mode 토글
- 고정 사용자 프로필
- 고정 오늘 상태
- 고정 등급 결과 선택
- AI 실패 시 fallback 카드 즉시 표시
- 친구방/전체방 더미 피드 내장

## 8. 핵심 사용자 루프

### 8.1 기본 루프

```text
앱 실행 → 오늘의 픽업 확인 → 상태 선택 → 인간 가챠 뽑기 → AI 인간 카드 → 미션 수행 → 뱃지/실패 칭호 획득 → 친구방 공유 → 다음 날 재방문
```

### 8.2 기대감 루프

```text
픽업 인간 확인 → 확률 버프 확인 → 캡슐 연출 → 등급 실루엣 공개 → 희귀 후보 등장 → 친구방 자동 공개
```

### 8.3 실패 콘텐츠 루프

```text
높은 등급 후보 획득 → 미션 실패 → 실패 칭호 획득 → 자아 파편 지급 → 친구방 리액션 → 다음 날 재도전
```

## 9. 90초 시연 스토리보드

### 9.1 데모 전제

데모 사용자:

```text
닉네임: 현기
유형: 취준 인간
자주 실패하는 것: 코테, 운동, 답장
말투: 적당히 킹받게
오늘 상태: 잠 부족, 카페인 필요, 코테 준비 중
```

데모 결과는 `SR` 또는 `SSR`로 고정한다. `LEGEND`는 너무 과하면 조작처럼 보일 수 있으므로 예비 시나리오로만 사용한다.

### 9.2 90초 플로우

| 시간 | 화면 | 보여줄 것 | 발표 멘트 |
|---:|---|---|---|
| 0~10초 | Home | 오늘의 픽업 인간, 확률 버프, 뽑기 버튼 | "자기관리 목표를 직접 쓰는 대신 인간 등급과 미션을 뽑습니다." |
| 10~20초 | 상태 선택 | 잠 부족, 카페인 필요, 코테 준비 중 선택 | "오늘 상태 몇 개만 고르면 AI가 미션을 맞춥니다." |
| 20~35초 | Draw Animation | 인간성 스캔, 등급 실루엣, SR 공개 | "가챠처럼 결과를 기다리게 만들었습니다." |
| 35~50초 | Result Card | SR 계획적인 취준 인간 후보, 미션, 실패 칭호 | "높은 등급이 나와도 미션을 해야 확정됩니다." |
| 50~62초 | Mission Complete | 완료 버튼, 뱃지 획득 | "작은 행동 하나를 보상으로 연결합니다." |
| 62~78초 | Friend Room | 자동 공유 포스트, 리액션 | "친구방에서는 서로의 인간 등급을 구경하고 놀릴 수 있습니다." |
| 78~90초 | Whole Room / Summary | 오늘 전체 유저 N급 52%, 가장 많이 실패한 미션 | "실패까지 콘텐츠가 되기 때문에 다시 열 이유가 생깁니다." |

### 9.3 데모 카드 예시

```text
SR 계획적인 취준 인간 후보

설명:
계획표를 열었고, 아직 도망가지 않았습니다.
이 정도면 오늘의 사회적 생존 가능성이 있습니다.

오늘의 미션:
코테 문제를 다 풀지 않아도 좋으니 풀이를 15분만 읽으세요.

성공 보상:
도망은 안 간 사람 뱃지

실패 칭호:
SR인 척한 사람
```

### 9.4 예비 임팩트 카드

발표 분위기가 약할 때 사용할 SSR 카드:

```text
SSR 오전 부팅 성공자 후보

설명:
오전에 움직였다는 제보가 들어왔습니다.
현재 인간성 서버가 혼란스러워하고 있습니다.

오늘의 미션:
이력서 한 문단을 열고, 딱 5줄만 고쳐 쓰세요.

성공 보상:
오전 부팅 성공자 뱃지

실패 칭호:
부팅 화면에서 멈춘 사람
```

## 10. MVP 범위

### 10.1 P0: 데모 필수

| 기능 | 이유 | 구현 기준 |
|---|---|---|
| Demo User 진입 | 로그인 지연 제거 | 앱 실행 후 바로 홈 진입 가능 |
| 온보딩 선택 | 개인화 근거 | 4개 질문 이하, 버튼 선택만 |
| 오늘 상태 선택 | AI 개인화 입력 | 1~3개 선택 |
| 오늘의 픽업 인간 | 뽑기 기대감 | 홈에서 2~3개 노출 |
| 확률 버프 표시 | 게임성 인지 | 실제 계산보다 시각 피드백 우선 |
| 인간 가챠 뽑기 | 핵심 기능 | 버튼 1개로 연출 시작 |
| 뽑기 연출 | 시연 임팩트 | 3단계 이상 텍스트/카드 전환 |
| AI 인간 카드 생성 | 기술 차별점 | JSON 출력, 5초 내 실패 시 fallback |
| 미션 완료 버튼 | 루프 완성 | 완료 즉시 뱃지 화면 |

## 11. 핵심 기능 요구사항

## 11.1 Demo User / Google 로그인

### 목적

시연에서는 로그인으로 시간을 쓰지 않고 바로 핵심 재미를 보여준다. 제품 MVP에서는 Google 로그인으로 사용자 기록을 저장한다.

### P0 요구사항

- 데모 계정은 `현기 / 취준 인간` 프로필을 기본값으로 가진다.
- 데모 계정은 Firestore 없이 로컬 상태만으로도 전체 플로우를 완료할 수 있다.

### P1 요구사항

- 사용자는 Google 계정으로 로그인할 수 있다.
- 최초 로그인 사용자는 온보딩으로 이동한다.
- 기존 사용자는 홈으로 이동한다.
- Firebase Auth UID 기준으로 사용자 문서를 생성한다.

## 11.5 인간 가챠 뽑기

### 목적

서비스의 핵심 재미를 만든다.

### 결과 구성

- 등급
- 인간 타입
- 후보 상태
- 설명
- 오늘의 미션
- 성공 뱃지
- 실패 칭호
- 자아 파편 보상
- 공유 문구

### 등급 확률

| 등급 | 기본 확률 | 예시 |
|---|---:|---|
| N | 45% | 침대 협상가 |
| R | 30% | 카페인 의존형 인간 |
| SR | 15% | 계획적인 취준 인간 |
| SSR | 8% | 오전 부팅 성공자 |
| UR | 1.8% | 빨래까지 한 사람 |
| LEGEND | 0.2% | 지원서 쓰고 코테 풀고 씻은 사람 |

### 핵심 정책

- 뽑기 결과는 `후보`다.
- 미션 성공 시 뱃지가 확정된다.
- 미션 실패 시 실패 칭호와 자아 파편을 받는다.
- 데모 모드에서는 등급 결과를 고정할 수 있다.

### 수용 기준

- 결과 공개 전 최소 3단계 연출이 있다.
- 등급이 먼저 공개되고, 인간 타입과 미션이 순차 공개된다.
- 높은 등급 후보는 친구방 피드에 강조 표시된다.

## 11.6 뽑기 연출

### 목적

단순 버튼 클릭을 기다림과 기대감이 있는 이벤트로 바꾼다.

### 연출 단계

1. 자아 캡슐 흔들림
2. 인간성 스캔 문구 노출
3. 등급 실루엣 또는 색상 공개
4. 등급 공개
5. 인간 타입 공개
6. 미션 공개
7. 성공 보상과 실패 칭호 공개

### 연출 문구

```text
인간성 스캔 중...
어제의 양심 잔여량 확인 중...
사회적 생존 가능성 계산 중...
침대와의 협상 기록 분석 중...
오늘의 인간 타입 확정!
```

### 수용 기준

- 전체 연출은 6초를 넘기지 않는다.
- Skip은 제공하지 않아도 된다.
- 발표용으로 결과 공개 순간의 시각 강조가 가장 중요하다.

## 11.7 AI 인간 카드 생성

### 목적

사용자 프로필, 오늘 상태, 뽑힌 등급을 바탕으로 매일 다른 인간 카드를 생성한다.

### AI 입력

```json
{
  "nickname": "현기",
  "persona": "취준 인간",
  "mainStruggles": ["코테", "운동"],
  "difficulty": "normal",
  "tone": "roast",
  "todayMood": ["잠 부족", "카페인 필요", "코테 준비 중"],
  "drawnGrade": "SR"
}
```

### AI 출력

```json
{
  "gradeTitle": "SR 계획적인 취준 인간",
  "description": "계획표를 열었고, 아직 도망가지 않았습니다. 이 정도면 오늘의 사회적 생존 가능성이 있습니다.",
  "mission": "코테 문제를 다 풀지 못해도 좋으니 풀이를 15분만 읽으세요.",
  "successBadge": "도망은 안 간 사람",
  "failureTitle": "SR인 척한 사람",
  "failureMessage": "계획은 있었으나 실행 파일이 누락되었습니다.",
  "shareText": "오늘 나는 SR 계획적인 취준 인간 후보입니다."
}
```

### 프롬프트 초안

```text
너는 B급 자기관리 가챠 앱 '인간 가챠'의 카드 생성기다.

사용자 정보:
- 닉네임: {nickname}
- 사용자 유형: {persona}
- 자주 실패하는 것: {mainStruggles}
- 미션 난이도: {difficulty}
- 말투 강도: {tone}
- 오늘 상태: {todayMood}
- 뽑힌 등급: {drawnGrade}

조건:
- 사용자가 오늘 안에 실제로 할 수 있는 짧은 미션을 만든다.
- 목표를 직접 쓰지 않아도 되는 선택형 자기관리 경험에 맞춘다.
- B급이고 웃기되, 사용자를 혐오하거나 심하게 비하하지 않는다.
- 우울, 자해, 질병, 외모, 신체, 성별, 인종, 장애, 가족사를 놀리지 않는다.
- 실패 문구는 죄책감이 아니라 가벼운 농담이어야 한다.
- 결과는 JSON 형식으로만 응답한다.
- 각 필드는 60자 이내를 우선한다.

출력 형식:
{
  "gradeTitle": "",
  "description": "",
  "mission": "",
  "successBadge": "",
  "failureTitle": "",
  "failureMessage": "",
  "shareText": ""
}
```

### Fallback 카드

AI 호출 실패, JSON 파싱 실패, 네트워크 실패 시 즉시 아래 템플릿 중 하나를 사용한다.

```text
SR 계획적인 사람
설명: 계획표를 열었고 아직 도망가지 않았습니다.
미션: 오늘 할 일 중 가장 만만한 것 1개를 10분만 처리하세요.
성공 뱃지: 작심일일 생존자
실패 칭호: 계획만 세운 사람
```

### 수용 기준

- AI 응답은 5초 이내를 목표로 한다.
- 5초 초과 시 fallback을 먼저 보여주고, 재시도 버튼을 제공한다.
- AI 결과가 비어 있거나 길면 클라이언트에서 기본 문구로 보정한다.

## 11.8 미션 완료와 보상

### 목적

### P0 인증 방식

- 실패 처리 버튼

### P1 인증 방식

- 선택적 사진 첨부

### 성공 처리

- 뱃지 획득
- 자아 파편 지급

### 실패 처리

- 실패 칭호 획득
- 자아 파편 소량 지급
- streak 종료
- 친구방 포스트에 실패 상태 반영

### 수용 기준

- 완료 버튼 클릭 후 1초 안에 보상 화면이 나온다.
- 성공/실패 모두 보상 화면이 있어야 한다.
- 실패 화면은 사용자를 깎아내리지 않고 웃긴 기록처럼 보여야 한다.

## 11.9 친구방

### 목적

결과를 혼자 보고 끝내지 않고, 공유 가능한 장면으로 만든다.

### P0 정책

- 실제 친구 초대는 구현하지 않는다.
- 더미 친구 3~5명을 사용한다.
- 내 뽑기 결과가 피드 맨 위에 추가된 것처럼 표시한다.
- 리액션 버튼은 눌렀을 때 숫자만 증가해도 된다.

### 리액션 버튼

- 인정
- 구라 ㄴ
- 인간 승리
- 내 얘기임
- 지켜본다
- 실패 예약

### 피드 예시

```text
현기가 SR 계획적인 취준 인간 후보를 뽑았습니다.
성공하면 인간 승리, 실패하면 SR인 척한 사람입니다.

[인정 3] [구라 ㄴ 1] [지켜본다 4]
```

### 수용 기준

- 친구방은 발표에서 "이 앱이 공유될 수 있겠다"는 인상을 줘야 한다.
- 실시간성은 MVP에서 요구하지 않는다.
- 높은 등급은 색상, 테두리, 상단 고정으로 강조한다.

## 14. 뽑기 로직

### 14.1 기본 정책

- 하루 1회 무료 뽑기
- 한국 시간 기준 날짜 제한
- 자아 파편 사용 시 재뽑기 가능
- 전날 성공, streak, 친구 리액션에 따라 확률 보정
- Demo Mode에서는 forcedGrade가 있으면 확률 계산을 건너뛴다.

### 14.2 Dart 예시

```dart
enum HumanGrade {
  n,
  r,
  sr,
  ssr,
  ur,
  legend,
}

HumanGrade drawGrade({
  required int streak,
  required bool completedYesterday,
  required int cheerReactionCount,
  HumanGrade? forcedGrade,
}) {
  if (forcedGrade != null) {
    return forcedGrade;
  }

  final rates = <HumanGrade, double>{
    HumanGrade.n: 45.0,
    HumanGrade.r: 30.0,
    HumanGrade.sr: 15.0,
    HumanGrade.ssr: 8.0,
    HumanGrade.ur: 1.8,
    HumanGrade.legend: 0.2,
  };

  if (completedYesterday) {
    rates[HumanGrade.n] = rates[HumanGrade.n]! - 3;
    rates[HumanGrade.sr] = rates[HumanGrade.sr]! + 2;
    rates[HumanGrade.ssr] = rates[HumanGrade.ssr]! + 0.8;
    rates[HumanGrade.ur] = rates[HumanGrade.ur]! + 0.2;
  }

  if (streak >= 3) {
    rates[HumanGrade.n] = rates[HumanGrade.n]! - 2;
    rates[HumanGrade.r] = rates[HumanGrade.r]! + 1;
    rates[HumanGrade.sr] = rates[HumanGrade.sr]! + 1;
  }

  if (cheerReactionCount >= 3) {
    rates[HumanGrade.ssr] = rates[HumanGrade.ssr]! + 0.5;
    rates[HumanGrade.n] = rates[HumanGrade.n]! - 0.5;
  }

  final total = rates.values.reduce((a, b) => a + b);
  final random = Random().nextDouble() * total;

  double cumulative = 0;

  for (final entry in rates.entries) {
    cumulative += entry.value;
    if (random <= cumulative) {
      return entry.key;
    }
  }

  return HumanGrade.n;
}
```

## 15. 미션 난이도 정책

| 등급 | 미션 수준 | 예시 |
|---|---|---|
| N | 1분 이내 | 물 한 컵 마시기 |
| R | 3~5분 | 책상 위 물건 3개 치우기 |
| SR | 10~15분 | 코테 풀이 15분 읽기 |
| SSR | 20~30분 | 이력서 한 문단 수정 |
| UR | 행동 2개 조합 | 운동복 입고 10분 걷기 + 샤워 |
| LEGEND | 보스 미션 | 지원서 1개 제출 |

정책:

- 낮은 등급은 너무 초라하게 느껴지면 안 된다. 웃기지만 완료하기 쉬워야 한다.
- 높은 등급은 어렵지만 오늘 안에 끝낼 수 있어야 한다.
- 미션은 실패해도 사용자가 다시 앱을 켤 수 있는 정도의 부담이어야 한다.

## 21. 발표용 핵심 메시지

### 문제

> 자기관리 앱은 시작부터 목표를 쓰라고 해서 부담스럽고, 실패하면 다시 열기 싫어집니다.

### 해결

> 인간 가챠는 목표를 직접 쓰게 하지 않고, 오늘의 상태를 바탕으로 AI가 하루 미션을 뽑아줍니다.

### 차별점

> 사용자는 인간 등급 후보를 뽑고, 미션을 성공해야 뱃지를 확정합니다. 실패해도 칭호와 자아 파편이 남아 실패까지 콘텐츠가 됩니다.

### 데모 한 문장

> 인간 가챠는 갓생을 강요하는 자기관리 앱이 아니라, 오늘 하루 덜 망한 인간을 뽑는 B급 AI 자기관리 서비스입니다.

## 22. 최종 MVP 정의

인간 가챠 MVP는 사용자가 현재 상태를 선택하면 AI가 인간 등급, 오늘의 미션, 성공 뱃지, 실패 칭호를 생성하고, 사용자가 미션을 완료하거나 실패 처리한 뒤 친구방 피드에서 결과를 공유하는 Flutter 기반 B급 자기관리 가챠 서비스다.

시연 최우선 구현 포인트는 다음 5가지다.

1. 앱 실행 직후 인간 가챠의 세계관이 보인다.
2. 오늘 상태 선택만으로 개인화가 된다.
3. 뽑기 연출과 등급 공개가 가장 강한 장면이 된다.
4. AI 결과 카드가 웃기고 공유 가능해야 한다.
5. 데모 모드와 fallback으로 발표 중 실패하지 않는다.
